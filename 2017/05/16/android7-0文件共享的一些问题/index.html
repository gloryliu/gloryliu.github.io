android7.0文件共享的一些问题

发表于 2017-05-16 | 分类于 开发经验

在Android7.0的系统上，会遇到报如下错误。
Caused by: android.os.FileUriExposedException
原因是，安卓官方为了提高私有文件的安全性，面向 Android 7.0 或更高版本的应用私有目录被限制访问　(0700)。此设置可防止私有文件的元数据泄漏，如它们的大小或存在性.

传递软件包网域外的 file:// URI 可能给接收器留下无法访问的路径。因此，尝试传递 file:// URI 会触发 FileUriExposedException。分享私有文件内容的推荐方法是使用 FileProvider。

标签下面必须包含至少包含以下标签中的一个或者多个。
files-path:表示Context.getFilesDir()目录或者其子目录。

表示路径 : /data/data/com.liu.youyou/files/test1
cache-path:表示Context.getCacheDir()目录或者其子目录。

表示路径 : /data/data/com.liu.youyou/cache/test2
external-path:表示Environment.getExternalStorageDirectory()目录或者其子目录。

表示路径 : /storage/emulated/0/test3
external-files-path:表示Context.getExternalFilesDir(null)目录或者其子目录

表示路径 : /storage/emulated/0/Android/data/com.liu.youyou/files/test4
external-cache-path:表示Context.getExternalCacheDir()目录或者其子目录

表示路径 : /storage/emulated/0/Android/data/com.liu.youyou/cache/test5

总结注意点:

1、android:authorities=”com.liu.youyou.fileprovider”,这里的authorities值为包名加”.fileprovider”
2、FileProvider.getUriForFile(getContext(), “com.liu.youyou.fileprovider”, xxxFile);第二个参数必须和provider中的authorities的值完全一样。
3、这里的name可以随便，path不要写父文件，比如path=”/storage/emulated/0/good”,这样写是不行的，写个”good/“就ok。
